# Личная медкарта

Личное приложение для отслеживания здоровья: визиты к врачу, прививки, анализы, УЗИ, витамины, вес и рост — по датам, с вложенными фото и документами, плюс подсказки по чек-апу в зависимости от возраста и пола. Поддерживает до 5 отдельных профилей (например, для всей семьи).

Все данные и фотографии хранятся **только в вашем браузере, на вашем устройстве** — никуда не отправляются, никому не видны. Это значит, что приложению не нужен ни сервер, ни база данных, ни платные сервисы — достаточно один раз опубликовать один файл на GitHub Pages.

---

## 1. Как опубликовать сайт (5–10 минут, без программирования)

Понадобится только аккаунт на [github.com](https://github.com) (бесплатный).

### Шаг 1. Создайте репозиторий
1. Зайдите на [github.com/new](https://github.com/new)
2. Название репозитория — любое, например `medcard` (латиницей, без пробелов)
3. Выберите **Public** (для бесплатного GitHub Pages репозиторий должен быть публичным)
4. Нажмите **Create repository**

### Шаг 2. Загрузите файл
1. На странице репозитория нажмите **Add file → Upload files**
2. Перетащите файл `index.html` из этого набора в окно браузера
3. Внизу нажмите **Commit changes**

### Шаг 3. Включите GitHub Pages
1. Откройте вкладку **Settings** репозитория
2. Слева выберите **Pages**
3. В разделе **Branch** выберите `main` и папку `/ (root)`, нажмите **Save**
4. Подождите 1–2 минуты и обновите страницу — вверху появится ссылка вида:
   `https://ваш-логин.github.io/medcard/`

Это и есть постоянный адрес вашего приложения. Он не изменится и продолжит работать без вашего участия.

### Шаг 4 (по желанию). Добавить на экран телефона
Откройте ссылку в браузере телефона → меню браузера → «Добавить на главный экран» (Android) или «На экран Домой» (iOS, через кнопку «Поделиться» в Safari). Значок появится как у обычного приложения.

## 2. Как пользоваться

- **Главная** — переключение между профилями (до 5), карточка активного профиля, сетка категорий с количеством записей (для «Вес и рост» — сразу последний результат).
- **Дневник** — лента всех записей по датам, с фильтром по категории. Кнопка «+» вверху — быстрое добавление.
- **Чек-ап** — список того, что стоит проверять в вашем возрасте (общие международные ориентиры, не привязаны к стране), с пометкой, что уже сделано, а что просрочено.
- **Профиль** — имя, дата рождения, пол; переключатель и добавление профилей; кнопки экспорта/импорта резервной копии.

## 3. Важно про данные и резервные копии

Раз данные хранятся только в браузере, это значит:
- Если вы очистите историю/данные браузера — записи исчезнут.
- Если откроете сайт в другом браузере или на другом устройстве — там будет пусто (это не облако, а именно локальное хранилище).

Поэтому во вкладке **Профиль** есть кнопка **«Скачать резервную копию»** — она сохраняет вообще всё (все профили, записи, фото) в один файл. Рекомендуется скачивать её время от времени, а через кнопку **«Загрузить резервную копию»** можно всё восстановить — например, после смены телефона.

## 4. Как изменить что-то самой (без программирования)

Если захотите подправить список вариантов для «Витаминов», добавить пункт в рекомендации чек-апа или изменить цвет категории — не нужно ничего переустанавливать. В репозитории на GitHub откройте `index.html`, нажмите значок карандаша (Edit) в правом верхнем углу — откроется редактор прямо в браузере. Используйте поиск (Ctrl+F / Cmd+F) по названию нужного блока:

- `TYPE_META` — названия, иконки и цвета категорий
- `TITLE_SUGGESTIONS` — варианты автозаполнения (например, список витаминов)
- `CHECKUP_RULES` — пункты чек-апа: возраст, периодичность, для кого

Поменяйте нужный текст между кавычками, внизу страницы нажмите **Commit changes** — через минуту изменения появятся на сайте автоматически.

## 5. Идеи на будущее

Если понадобится — можно добавить график изменения веса во времени, напоминания о ближайших чек-апах, или PWA-версию для offline-использования. Просто попросите в чате с Claude, приложив этот же файл или ссылку на репозиторий.
